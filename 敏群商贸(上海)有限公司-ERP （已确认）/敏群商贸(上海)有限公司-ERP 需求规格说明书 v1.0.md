# 面料 ERP 需求规格说明书（v1.0）

> 文档说明：本规格说明书基于已确认需求文档及原软件 `E:\360MoveData\Users\Administrator\Desktop\HSTIP_SHMQ` 整理补充。原软件为 HSTIP/环思类桌面 ERP，主要参考其基础资料、面料资料、样品选样、样品库存、查询、导出、打印等业务能力。本轮实施以“主进销存、辅管理”为目标，重点保留客户已确认页面及相关核心流程。

---

## 1 项目概述

### 1.1 项目目标

建设一套面向纺织面料样品贸易场景的 ERP 系统，实现从面料资料录入、客户选样、样品查询、表格导出、标签打印到基础权限管理的业务闭环。

系统需参考原 HSTIP_SHMQ 软件中的相关模块，但不做完整旧系统复刻，重点迁移和优化以下业务：

- 面料类别维护
- 面料资料维护
- 供应商维护
- 客户资料维护
- 客户选样管理
- 客户选样查询
- 面料查询
- 样品库存相关能力（样品入库、样品出库、样品库位、库存查询，按实际业务范围裁剪）
- 标签打印与二维码扫码
- 表格预览与 Excel 导出
- 账号、角色、权限、数据字典等系统基础能力

### 1.2 关键约束

- 账号模式：双角色（管理员、员工），数据互通，互不踢人；员工账户无法查看「供应商维护」和「客户资料维护」。
- 标签打印：需兼容 Argox 立象 CP-2140M/3140，铜版纸 70mm×40mm。
- 不做范围：销售管理、采购管理、财务管理、电子审批不在本轮实施。

## 2 角色与权限

### 2.1 角色定义

#### 管理员账户

管理员可访问全部本轮实施模块：

- 面料类别维护
- 面料资料维护
- 供应商维护
- 客户资料维护
- 客户选样管理
- 客户选样查询
- 面料查询
- 样品库存相关页面
- 标签打印
- 数据字典
- 用户账号管理
- 操作日志查看

#### 员工账户

员工账户可访问日常业务功能，但不可访问敏感基础资料：

- 可访问：面料类别维护、面料资料维护、客户选样管理、客户选样查询、面料查询、标签打印、样品库存查询。
- 不可访问：供应商维护、客户资料维护。
- 敏感字段：供应商、成本、采购价等字段是否对员工可见需最终确认；默认按权限隐藏。

### 2.2 登录与会话

- 同一账户允许多端同时登录，不互相顶掉。
- 多账号之间数据实时互通。
- 建议保留操作日志，用于追踪新增、修改、删除、打印、导出等动作。
- 建议支持修改密码、重置密码、启用/停用账号。

## 3 功能模块

## 3.1 前期管理

### 3.1.1 面料类别维护

#### 功能目标

维护面料分类资料，用于面料资料归类、查询筛选、选样展示。

#### 功能清单

- 新增类别
- 修改类别
- 删除类别
- 启用/停用类别
- 类别树展示
- 上级/下级分类维护
- 关键字查询

#### 字段要求

| 字段 | 要求 | 说明 |
|---|---|---|
| 类别编码 | 必填、唯一 | 可手动输入或系统生成 |
| 类别名称 | 必填 | 分类显示名称 |
| 上级类别 | 可选 | 支持多级分类 |
| 排序号 | 可选 | 控制展示顺序 |
| 是否启用 | 必填 | 停用后不可新增引用 |
| 备注 | 可选 | 补充说明 |

#### 校验规则

- 类别编码不可重复。
- 已被面料资料引用的类别不允许直接删除，可改为停用。
- 停用类别不影响历史数据展示。

### 3.1.2 面料资料维护

#### 功能目标

维护面料基础信息、规格信息、成本信息、产品图片和标签打印所需信息，是系统最核心的主数据。

#### 功能清单

- 新增面料资料
- 修改面料资料
- 删除/停用面料资料
- 上传产品图片
- 图片预览/放大
- 多条件查询
- 列表分页
- Excel 导入/导出（建议）
- 标签预览/打印入口
- 二维码生成入口
- 相似图片检索入口（待确认是否首期）

#### 字段要求

| 字段 | 要求 | 说明 |
|---|---|---|
| 面料编码 / Item No. | 必填、唯一 | 标签二维码主要识别字段 |
| 面料名称 | 必填 | 中文或英文名称 |
| 面料类别 | 必填 | 来源于面料类别维护 |
| 成分 / Composition | 可选 | 如 Cotton、Polyester 等 |
| 组织 / Construction | 可选 | 标签与表格中展示 |
| 幅宽 / Width | 可选 | 如 57/58 等 |
| 克重 / Weight | 可选 | 如 320G/M2 |
| 规格描述 | 可选 | 规格不完整时可手动补充 |
| 颜色 / 色号 | 可选 | 供查询和选样使用 |
| 单位 | 可选 | 米、码、公斤等 |
| 成本价 | 可选、权限控制 | 是否员工可见待确认 |
| 销售价/参考价 | 可选 | 如客户选样表需展示可配置 |
| 供应商 | 可选、管理员可见 | 来源于供应商维护 |
| 库位 | 可选 | 来源于样品库位 |
| 库存数量 | 可选 | 由入库/出库汇总或手动维护 |
| 产品图片 | 可选 | 支持单图/多图，列表中显示缩略图 |
| 备注 | 可选 | 普通备注 |
| 标签备注 | 可选 | 默认打印备注，可被临时备注覆盖或追加 |
| 状态 | 必填 | 启用/停用 |
| 创建人/创建时间 | 系统生成 | 用于追溯 |
| 修改人/修改时间 | 系统生成 | 用于追溯 |

#### 图片要求

- 支持上传产品图片，至少支持 JPG、PNG、WEBP。
- 列表中支持缩略图展示。
- 详情页支持图片预览、放大、删除、替换。
- 批量选样导出时可选择是否带图。
- 建议限制单张图片大小，避免导出和查询过慢。

#### 查询条件

- 面料编码
- 面料名称
- 面料类别
- 成分
- 规格关键字
- 色号/颜色
- 供应商（管理员可见）
- 是否有图片
- 状态
- 创建时间范围

#### 业务规则

- 面料编码必须唯一。
- 停用面料不允许新增选样，但历史选样记录仍可查看。
- 有规格显示具体规格，无规格显示 `/`。
- 员工账户默认隐藏供应商、成本等敏感信息。

## 3.2 样品管理

### 3.2.1 客户选样管理（批量处理）

#### 功能目标

支持业务人员为某个客户批量选择样品，形成选样清单，并可进行表格预览、Excel 导出和批量标签打印。

#### 原系统参考

参考原软件中的：

- `HS_Trade_MM_SAMPLE_frmmmSampleChoose.dll`：客户选样
- `HS_Trade_MM_SAMPLE_frmmmSampleChooseQuery.dll`：客户选样查询
- `HS_Trade_MM_SAMPLE_frmpbSampleCustomer.dll`：样品客户资料
- 用户配置中的 `TfrmmmSampleChoose`、`TfrmmmSampleChooseQuery`

#### 页面布局

建议页面包含：

1. 客户信息区
   - 选择客户
   - 联系人/地区展示
   - 选样日期
   - 操作人
2. 商品录入区
   - 手动输入面料编码
   - 扫码枪输入框
   - 查询选择面料按钮
3. 选样明细表
   - 已选面料列表
   - 图片缩略图
   - 规格、成本、备注等字段
4. 操作区
   - 保存选样单
   - 预览表格
   - 导出 Excel
   - 打印标签
   - 清空明细

#### 添加样品方式

##### 手动输入编码

- 用户输入面料编码。
- 系统校验面料是否存在。
- 存在则加入选样明细。
- 不存在则提示“未找到该面料编码”。
- 重复添加时提示是否继续添加或自动合并数量。

##### 扫描二维码

- 扫码枪扫描标签二维码。
- 二维码内容解析为面料编码或面料详情 URL。
- 系统自动定位面料资料并加入选样明细。
- 扫码后输入框自动清空并等待下一次扫码。

##### 查询选择

- 点击选择面料。
- 弹出面料查询窗口。
- 支持按编码、名称、类别、规格、图片等筛选。
- 勾选多条后加入选样明细。

#### 选样明细字段

| 字段 | 说明 |
|---|---|
| 序号 | 系统生成，可调整顺序 |
| 面料编码 | 来自面料资料 |
| 面料名称 | 来自面料资料 |
| 成分 | 来自面料资料 |
| 组织/规格 | 来自面料资料 |
| 幅宽 | 来自面料资料 |
| 克重 | 来自面料资料 |
| 颜色/色号 | 来自面料资料 |
| 成本/价格 | 可配置是否展示 |
| 图片 | 缩略图，可预览 |
| 选样数量 | 默认 1，可调整 |
| 临时备注 | 当前选样或打印时使用 |

#### 表格预览与导出

客户已确认：表格显示选项改为多选方式。

可选项：

- 规格
- 成本
- 图片

规则：

- 选中“规格”则展示完整规格字段。
- 未选中“规格”则仅展示基础字段。
- 选中“成本”则展示成本/价格字段。
- 未选中“成本”则隐藏成本/价格字段。
- 选中“图片”则在表格尾部展示图片列。
- 未选中“图片”则不展示图片。
- 规格不完整时，缺失字段显示 `/`。
- 导出 Excel 时需优化排版，避免大量细碎小单元格。

#### 选样单保存

保存后生成客户选样记录，用于后续查询。

保存内容包括：

- 选样单号
- 客户信息快照
- 选样日期
- 操作人
- 选样明细快照
- 导出/打印记录（建议）

## 5 业务流程

### 5.1 面料资料建档流程

1. 管理员或员工进入面料资料维护。
2. 新增面料资料。
3. 录入编码、类别、成分、规格、幅宽、克重等信息。
4. 上传产品图片。
5. 保存。
6. 系统生成二维码所需数据。
7. 可进入标签预览并打印标签。

### 5.2 客户选样流程

1. 进入客户选样管理。
2. 选择客户。
3. 通过手动编码、扫码枪或查询选择添加面料。
4. 系统将面料加入选样明细。
5. 勾选导出显示项：规格、成本、图片。
6. 预览客户选样表。
7. 导出 Excel 或打印标签。
8. 保存客户选样记录。

### 5.3 扫码添加流程

1. 用户新建或打开一张未作废的客户选样单，并将光标定位在扫码输入框。
2. 扫码枪以键盘输入模式扫描标签二维码，并以 Enter 作为结束符。
3. 前端接收完整扫码内容，去除首尾空格后按二维码规则解析面料编码。
4. 系统按面料编码查询启用状态的面料资料。
5. 查询成功后自动加入选样明细并显示成功反馈，输入框清空且重新聚焦，等待下一次扫描。
6. 若条码为空、格式不支持、面料不存在或已停用，显示明确原因且不新增明细。
7. 同一面料重复扫描时，按页面设定的合并或重复添加规则处理，并向用户展示结果。

### 5.4 标签打印流程

1. 从面料查询或客户选样进入标签打印。
2. 系统加载标签模板。
3. 用户选择打印条目和份数。
4. 用户可输入临时备注。
5. 预览标签。
6. 打印到 Argox 标签打印机。
7. 记录打印日志。

### 5.5 样品入库/出库流程

1. 入库：选择面料 → 输入数量和库位 → 保存 → 增加库存。
2. 出库：选择面料 → 输入数量和用途 → 校验库存 → 保存 → 减少库存。
3. 查询：按面料、库位、时间查询库存和流水。

## 6 非功能需求

### 6.1 性能

- 常规列表查询需支持分页。
- 图片需使用缩略图展示，避免列表加载原图。
- Excel 导出大量图片时需有等待提示。
- 扫码添加需快速响应，避免连续扫码丢失。

### 6.2 可用性

- 界面应比旧桌面软件更清晰，但保留原有业务习惯。
- 表单字段需区分必填、只读、可编辑状态。
- 删除类操作需二次确认。
- 保存成功、导出成功、打印成功需有明确提示。

### 6.3 安全

- 密码加密存储。
- 接口需校验登录状态和权限。
- 员工不可通过接口访问客户、供应商、成本等无权限数据。
- 导出与打印记录需保留日志。

### 6.4 兼容性

- 支持 Chrome、Edge 浏览器。
- 支持 USB 扫码枪键盘输入模式。
- 支持 Argox 立象 CP-2140M/3140 标签打印。

### 6.5 跨地区访问与部署

- 系统须在中国内地、台湾、香港稳定使用；台北、香港的日常查询、保存、扫码、导出与打印操作应保持正常可用速度。
- 生产环境部署方案须选择可同时覆盖上述地区的网络与云资源，并启用 HTTPS。
- 图片文件应使用适合跨地区访问的对象存储与加速方案，列表仅加载缩略图。
- 上线前须以中国内地、台北、香港三个网络环境分别执行登录、查询、保存、图片预览、扫码添加、Excel 导出及标签打印联调。
- 跨地区网络异常时，系统应明确提示请求失败；不得将未保存的数据误提示为已保存。
